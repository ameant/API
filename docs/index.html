Glossaire

API et Documentation

API : Application Programming Interface. Un ensemble de définitions et de protocoles pour construire et intégrer des logiciels d'application.

Token : Un jeton d'authentification est une chaîne générée par le serveur pour vérifier l'identité de l'utilisateur et accorder l'accès à certaines ressources.

Middleware : Une fonction qui reçoit les objets de requête et de réponse, ainsi qu'une référence à la fonction middleware suivante dans le cycle de requête-réponse de l'application.

Authentification et Sécurité

JWT : JSON Web Token. Un standard ouvert pour la création de jetons d'accès qui permettent des échanges sécurisés d'informations entre parties.

Technologies et Concepts

MongoDB : Une base de données NoSQL orientée documents, utilisée pour stocker les données de votre application.

Express.js : Un framework web pour Node.js, utilisé pour construire des applications web et des APIs.

Node.js : Un environnement d'exécution JavaScript qui permet de créer des applications côté serveur.

REST : Representational State Transfer. Un style d'architecture pour la conception de réseaux distribués et des API.

JSON : JavaScript Object Notation. Un format léger d'échange de données facile à lire et à écrire pour les humains et facile à analyser et à générer par les machines.

Développement et Outils

JSDoc : Un outil utilisé pour générer de la documentation à partir de commentaires dans le code source JavaScript.

Nodemon : Un outil qui aide à développer des applications basées sur Node.js en redémarrant automatiquement l'application lorsque des modifications dans les fichiers du répertoire sont détectées.

Git : Un système de contrôle de version distribué utilisé pour suivre les modifications dans le code source pendant le développement logiciel.

GitHub : Une plateforme de développement où les développeurs peuvent héberger et gérer leur code, et collaborer avec d'autres.
